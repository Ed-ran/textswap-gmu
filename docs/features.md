# Feature Priorities — MoSCoW

## P1: Critical (MVP — Must Have)
- .edu account + basic verification
- Create listing (title, course/professor, condition, price, photos)
- Browse & basic search (title/course)
- Escrow flow (hold → confirm → release; refund path)
- Buyer receipt confirmation
- Minimal dispute path (flag + manual resolve)
- Order status (pending / in-escrow / completed / refunded)
- Basic coordination (in-app chat OR pickup-slot selection)
- Notifications for key events
- Terms/acceptable-use + light audit log

## P2: Important (Should Have)
- Ratings/reviews
- Price suggestions (past campus sales)
- Quick course tags
- Locker/venue pickup integration
- Saved searches / favorites
- Basic admin tools (metrics, manual adjustments)

## P3: Nice to Have (Could Have)
- Digital/open-access eBooks (partnered only)
- Shipping label integration (USPS/UPS)
- Ambassador tools (promo codes, department seeding)
- Seller analytics (time-to-sale, price bands)
- Referral credits & promos

## P4: Future Enhancements
- Smart-contract release logs
- Real-time updates (WebSocket)
- Hardened auth/SSO
- Multi-campus partner API
